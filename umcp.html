// the problem

AI systems forget.
Every. Single. Session.

⟳

Stateless by Design

Every AI session starts from zero. You re-explain the same context, same decisions, same preferences — every time.

Claude doesn't know what Cursor learned. Cursor doesn't know what ChatGPT decided. Each tool is an island.

// the protocol

Three operations.
Infinite context.

UMCP defines a minimal, transport-agnostic interface any AI tool can implement.

POST
/memory

Store a new memory node with type, content, and optional graph relations.

POST
/context

Query relevant memory for a task. Returns ranked nodes by hybrid score.

PATCH
/memory/:id

Update an existing node. Timestamps and graph edges preserved.

// Any AI tool — one unified interface
const client = new UMCPClient({
  storage: new JsonFileStorage()
});

// Claude stores a decision
await client.store({
  type: "decision",
  content: "Switched to PostgreSQL for relational queries"
});

// Cursor queries in a new session — hours later
const ctx = await client.query({
  task: "build authentication system",
  limit: 5
});

// → "Switched to PostgreSQL..." ✓

// retrieval algorithm

Context ranked by
what actually matters.

Score(n) = α·S + β·R + γ·G
S

Semantic Similarity

Jaccard similarity between query tokens and node content. Finds the most relevant memories for the current task.

R

Recency Decay

Exponential decay with 7-day half-life. Recent context ranks higher — stale decisions fade naturally.

G

Graph Proximity

Nodes related to other high-scoring nodes rank higher. Context follows your project's structure.

// storage backends

Two backends.
One interface.

VERSION 1

JSON File

Zero dependencies. Human-readable. Local-first.

  • No install required
  • Open the file, read your memory
  • Easy to backup or version-control
  • Works offline, always
  • Full file scan on every query
  • Not ideal for 10k+ nodes
Best for: personal projects, CLI tools, prototypes
VERSION 2

SQLite

Production-grade. Indexed. Still local-first.

  • Fast indexed queries
  • Handles thousands of nodes
  • Filter by type, date range
  • ACID transactions
  • Requires better-sqlite3
  • Slightly more setup
Best for: production use, multi-tool setups
